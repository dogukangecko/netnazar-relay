▍What is this?

NetNazar runs entirely on your device. For optional remote monitoring — watching your home network from outside and reaching device web UIs — it uses a relay that you self-host. This repository is exactly that server, so you can read, audit, build and run it yourself.

netscan-relay

The relay server (Rust · axum + PostgreSQL). Receives inventory/metrics from your agent, serves them to your app, dispatches away-notifications to your own channels, keeps a hash-chained audit log.

netscan-proto

The documented wire types exchanged between the agent and the relay — the protocol, in the open.

Web panel

A read-only panel and a reverse tunnel to device web UIs, with an SSRF guard restricting the agent to private IPv4.

▍Why self-host?

Zero data-custody

We never receive your scan data. The relay you run does — and only that.

No third party

Away-notifications go through channels you configure: your ntfy, your Telegram bot, your webhook.

Auditable

Read every line. Verify there is no telemetry, no phone-home, no hidden sink.

▍Quick start (Docker)

Clone, set a database password, and bring it up. The relay applies its schema migrations automatically on startup and listens on :8080.

# clone & enter
git clone https://github.com/dogukangecko/netnazar-relay.git
cd netnazar-relay

# set a real DB password in production
export NETSCAN_DB_PASSWORD="$(openssl rand -hex 16)"

# PostgreSQL + relay on :8080
docker compose up -d --build

Point your agent's NETSCAN_RELAY_URL at it, then log in from the NetNazar app's remote view. The same binary doubles as an admin CLI:

netscan-relay enroll                          # agent id + push key
netscan-relay create-user --email you@example.com --password '...'
netscan-relay add-channel --kind ntfy ...     # wire an away-notification channel

▍How it works

your network your server your phone ┌──────────────┐ push (HTTPS) ┌────────────────┐ read ┌──────────────┐ │ NetNazar │ ─────────────▶ │ netscan-relay │ ──────▶ │ NetNazar app │ │ agent (scan) │ inventory, │ + PostgreSQL │ login, │ (remote view)│ │ │ metrics │ │ events │ │ └──────────────┘ └────────────────┘ └──────────────┘ outbound-only REST + WebSocket (axum) opaque session token

The agent pushes inventory and metrics (authenticated with an agent key). The app logs in for an opaque, hashed, expiring session token and reads networks, devices, metrics, outages, events and the audit log. A reverse tunnel lets the app reach a device's web UI through the relay. Multi-tenant from day one.

▍Configuration

VariableRequiredPurpose
NETSCAN_DATABASE_URL✅PostgreSQL connection string

Secrets (DB URL, channel tokens) are read from the environment — never hardcoded.

Open-core. The relay and protocol are open source (AGPL-3.0) so the component running on your server is fully auditable. The NetNazar app and the scanning engine are proprietary and ship through the App Store / Google Play / signed desktop builds.
